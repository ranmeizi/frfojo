# ExecEnhanced — 乐观棋盘 + pushhandle 竞速方案

面向**自动化 2048 竞速**：优先延迟，可接受**约 1～2 步**内模型输入与真实局面因「随机新块」产生偏差；后期在棋盘上出现**大数（如 max ≥ 1024 或与阈值组合）**后，**关闭前端按 2048 规则的乐观合成**，仅依赖 **grid 权威更新**（首次读盘 + `pushhandle` 回执等）。

---

## 1. 双轨状态

| 状态 | 含义 | 更新时机 |
|------|------|-----------|
| **权威 `grid_authoritative`** | 与客户端一致、可喂模型 | **首次**成功读到的 grid；每次 **`pushhandle` 成功回执**（或协议规定的等价「局面已落盘」信号） |
| **乐观 `grid_optimistic`（可选）** | 仅用于 UI / 竞速中间态 | `grid_authoritative` + 选定方向 → 按与**游戏一致**的 2048 规则做**滑动与合并**；**不**生成随机新块（或仅 UI 占位，**不得**作为长期权威） |

---

## 2. 核心流程（竞速版）

1. 用**首次** grid 初始化权威状态。  
2. 用权威 grid 调用模型 → 得到最佳方向。  
3. **立刻**用 2048 规则在前端算出「合并后、生成随机块**前**」的棋盘，更新 UI（乐观）。  
4. **立即**发送 `pushhandle`（与方向一致），**串行**等待**完成回执**。  
5. 回执携带（或可解析出）**新权威 grid** → 更新 `grid_authoritative`，再进入步骤 2。  

目标：不必等 Lua/`grid.csv` 动画与写盘完成再进入下一轮决策，整体节拍缩短。

---

## 3. 已知不确定项：随机新块

- 标准 2048：每次**有效移动**后在随机空位生成 **2（常见约 90%）或 4（约 10%）**。  
- 前端在回执前**无法**预知位置与数值。  
- **若在回执到达前**用「无随机新块」的乐观盘再跑模型：与真局面可能差一格，**接受 1～2 步内有界误差**（竞速取舍）。  
- **回执到达后**必须以回执 grid 为准，避免误差累积。

---

## 4. 硬规则（实现时写死）

1. **误差预算**  
   - 约定例如：连续最多 **K=2** 次在权威 grid 未更新前允许用乐观盘驱动模型；超过 K 或检测到异常 → **强制同步**（等回执或读盘兜底）。

2. **大数阶段切换**  
   - 条件需**可测**：如 `max(tile) >= T`（例 T=1024）或与 `sum(大数)` 等组合。  
   - 一旦进入「仅权威」模式：**禁止**再用乐观合并结果参与模型输入；**下一拍**只使用回执/读盘后的 grid，避免切换边界半盘错误。

3. **无效移动**  
   - 若客户端对无效滑动**不推进局面、不生成新块**，前端规则必须与之一致；否则会出现「以为走了其实没走」，**不是** 1～2 步随机误差能自愈的问题。回执需能区分成功/拒绝。

4. **并发**  
   - `pushhandle` **严格串行**或带**单调序号/世代号**，保证回执与发送一一对应，禁止竞态。

5. **与游戏对齐**  
   - 合并顺序、滑动方向、是否「无位移也算无效」、生成概率与位置集合，须与客户端**逐项对齐**；否则乐观与回执会**系统性**偏差。

---

## 5. 推荐模型调用策略（二选一写进实现）

- **稳健**：每次仅在 `grid_authoritative` 更新后调用模型（回执到 → 更新 → 预测）。乐观只用于 UI。  
- **激进（本竞速方案）**：允许在回执前用乐观盘再预测 1 次（受 K 限制），**接受**棋质风险；回执到达后**覆盖**权威状态并丢弃与权威冲突的中间结论。

---

## 6. 验证与兜底

- 用固定种子或录屏局面对比：**前端合并（到生成前）** vs **回执 grid（去掉生成一步或全量对比）**，验证规则一致。  
- 保留**冷启动 / 纠错**路径：必要时读 `grid.csv` 或与回执对账，防止长时间漂移。

## 8. 当前代码落地（ExecEnhanced）

- **`libs/board2048.ts`**：`parseGridCsvText`、`maxTileInMatrix`、`applyMoveWithoutSpawn`（与 `libs/grid` 同源合并逻辑，**不**调用 `computerMove`）、`exportGridFromManager`。
- **`libs/filePushTransport.ts`**：`createFilePushHandleTransport` — `pushMove` **串行**；**await** `window.ffj_onPushHandle(direction, predictedBoard)`（第二个参数为「合并后、无随机块」矩阵），再以 `grid.csv` 的 `lastModified` 变化作为回执并 `readGrid`。
- **`libs/mockControlUnit.ts`**：测试挡板 `runMockControlUnitPush` — 在 **[0, 500) ms** 随机延迟后，将 `predictedBoard` 经随机 2/4 落子写入 `grid.csv`（`createWritable`）。
- **`ffjPushHandle.d.ts`**：`Window.ffj_onPushHandle` 类型声明（两参数）。
- **`index.tsx`**：选文件时使用 `showOpenFilePicker({ mode: 'readwrite' })`；开关「测试控制单元挡板」挂载/卸载挡板到 `window.ffj_onPushHandle`。
- 真实竞速可替换为自研控制单元，实现同样的 `(direction, predictedBoard) => void | Promise<void>` 即可。
